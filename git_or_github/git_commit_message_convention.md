# Git Commit Message Convention

## Git Commit Message Convention은 어떻게 가져가면 좋을까?

### 커밋 형식

[< `issue number` >] < `type` >: < `description` >

예시) [#1] feat: 로그인 기능 구현

### type 종류

| type     | 내용             |
|----------|----------------|
| feat     | 새로운 기능         |
| fix      | 버그 수정          |
| test     | 테스트 코드 추가 및 수정 |
| docs     | 문서 변경          |
| refactor | 코드 리팩터링        |
| chore    | 기타 작업          |
